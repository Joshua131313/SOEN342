## **Description**

This project is a Personal Task Management System to help an individual user organize and track tasks through the CLI. 

## **Objective**

The objective of this project is to  create a system based off of 3 partial system descriptions by analyzing the requirements with artifacts (Use-Case Diagram, UML Domain Model, SSDs, Fully-Dressed Scenarios, Operation Contracts, UML Class Diagram, Sequence Diagrams, OCL). Each iteration builds on top of the previos one, therefore rtifacts are updated every iteration to ensure systrem requirements are progressively satisfied.

## **Core features**
### 1. Task Details
The user can add numerous details to each task, notably a title, description, priority level, status, and due date. These details allow the user to track and organize their tasks. Task tags may also be attached to tasks to categorize and group them. 

### 2. Subtasks
Tasks may contain subtasks to allow complex tasks to be broken down into more manageable steps.

### 3. Projects
Tasks may be grouped together in a project.

### 4. Collaborators
External collaborators can be assigned to the user's tasks. These collaborators have a limit of tasks that they are allowed to be assigned to depending on their category (Senior, Intermediate, Junnior). 

### 5. Task Search and Filtering
The user may search tasks based on their title, tags, priority level, or status. Tasks may also be filtered based on their attributes. 

### 6. Importing Tasks from CSV
Tasks can be imported from a CSV file, meaning external task data may be loaded into the management system.

### 7. Exporting Tasks to Calendar
Tasks with deadlines can be exported to external calendar applications (ex.: Google Calendar).
