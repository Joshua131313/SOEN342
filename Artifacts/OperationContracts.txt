Operation Contracts:

Contract CO1  
Operation: createTask(title, description, priority, Status, dueDate)
Cross References: Use Case: Create Task
Preconditions: 
title exists.
If dueDate is not null, dueDate is not in the past.
Postconditions:
A Task instance t was created. (instance creation)
Task instance t has been added to the TaskDB. (Association formed)
Activity Entry ae instance is created. (Instance creation)

Contract CO2
Operation: updateTask(taskID, updatedValues)
Cross References: Use Case: Update Task 
Preconditions: 
Task instance t with taskID exists.
updatesValues contains at least one attribute to modify.
If title is a modified attribute, it is not getting modified to Null.
If status is a modified attribute, it is open, completed, or cancelled. 
Postconditions:
Task instance t has title updated if provided. (Attribute modification)
Task instance t has description updated if provided. (Attribute modification)
Task instance t has priority updated if provided. (Attribute modification)
Task instance t has dueDate updated if provided. (Attribute modification)
Task instance t has status updated if provided. (Attribute modification)
Task instance t’s association with Project instance p is updated. (Association modification)
Task instance t’s Tag instances are created or removed according to modifications. (Association creation/deletion)
Activity Entry ae instance is created. (Instance creation)

Contract CO3  
Operation: viewTasksList()
Cross References: Use Case: View Tasks List
Preconditions: 
At least one Task instance exists. 
Postconditions:
No state change. 

Contract CO4
Operation: sortTasksBy(attribute)
Cross References: Use Case: View Tasks List
Preconditions: 
At least one Task instance exists.
attribute belongs to the Task class.
Postconditions:
List of Task instance(s) ordered according to attribute selected. (No state change)

Contract CO5  
Operation: searchTasks(keywords)
Cross References: Use Case: Search Tasks 
Preconditions: 
keywords exists in at least one Task instance.
Postconditions:
List of Task instances that correspond to keywords is returned to the user. (No state change)

Contract CO6
Operation: createProject(title, description)
Cross References: Use Case: Create Project
Preconditions: 
title exists and is not empty.
Postconditions:
New Project instance p was created. (Instance creation)
Project instance p has been added to the ProjectsDB. (Association creation)
Activity Entry ae instance is created. (Instance creation)

Contract CO7
Operation: assignTasks(task[i], projectID)
Cross References: Use Case: Assign Tasks to Project
Preconditions: 
    - Project instance p with projectID exists
    - Each Task instance task[i] exists
Postconditions:
    - Each task[i] is associated with Project instance p. (Association formed)
    - The attribute task[i].project is set to p (Attribute modification)

Contract CO8
Operation: overwriteConfirm(decision)
Cross References: Use Case: Assign Tasks to Project
Preconditions: 
    - Task instance t is already associated with a project p1
    - Selected project p2 is different from current project p1
Postconditions:
    - Association between task t and with old project p1 is removed (Association broken)
    - Task t is associated with new project p2 (Association formed)
    - Project attribute t.project is set to the new project p2 (Attribute modification)

Contract CO9
Operation: createSubTask(title)
Cross References: Use Case: Create Subtask
Preconditions: 
    - Parent task instance t exists
    - Title exists (not empty)
Postconditions:
    - SubTask instance s is created (Instance creation)
    - Title attribute s.title is set to title (Attribute modification)
    - Status attribute is set to “open” (Attribute modification)

Contract CO10
Operation: assignSubTask(subtaskID, taskID)
Cross References: Use Case: Create Subtask
Preconditions: 
    - Task instance t exists with taskID
    - SubTask instance exists with subtaskID
    - S is not associated with a task
Postconditions:
    - Subtask s is associated with task t (Association formed)
    - Attribute s.parentTask is set to t (Attribute modification)
